1️⃣ *Automated tests require no maintenance once written.*
**Answer:** False
**Explanation:** Test scripts need updates as the application evolves.

2️⃣ *Explain the concept of flaky tests and how to mitigate them.*
**Answer:** Flaky tests produce intermittent failures due to timing, environment, or dependencies; mitigation includes isolating resources, adding retries, or stabilizing test data.
**Explanation:** Flakiness reduces confidence; proper design and environment control help.

3️⃣ *What is the difference between verification and validation?*
**Answer:** Verification checks if the product is built correctly (meets specifications), while validation checks if the right product is built (meets user needs).
**Explanation:** Verification is about process compliance; validation is about customer satisfaction.

4️⃣ *In agile, test automation should be introduced only at the end of the sprint.*
**Answer:** False
**Explanation:** Automation is built incrementally throughout the sprint.

5️⃣ *What does SDLC stand for and what are its main phases?*
**Answer:** Software Development Life Cycle; main phases are Requirements, Design, Implementation, Testing, Deployment, and Maintenance.
**Explanation:** SDLC provides a structured framework for planning and managing software projects.

6️⃣ *In the Waterfall model, each phase must be completed before the next begins.*
**Answer:** True
**Explanation:** Waterfall follows a sequential, phase-gate approach where phases do not overlap.

7️⃣ *What is the primary goal of agile testing?*
   - A) Find bugs early
   - B) Validate requirements
   - C) Ensure continuous feedback
   - D) All of the above
**Answer:** All of the above
**Explanation:** Agile testing aims for early defect detection and continuous feedback throughout development.

8️⃣ *Code reviews are considered a quality assurance activity.*
**Answer:** True
**Explanation:** Code reviews help catch defects early and improve code quality before deployment.

9️⃣ *What is a sprint in Scrum?*
   - A) A daily standup meeting
   - B) A time-boxed iteration of work
   - C) A retrospective event
   - D) A backlog refinement session
**Answer:** A time-boxed iteration of work
**Explanation:** A sprint is a fixed-length iteration (typically 1-4 weeks) during which a potentially shippable increment is created.

10️⃣ *Describe a risk‑based approach to manual test case selection.*
**Answer:** Prioritize test cases that cover high‑impact or high‑probability failure areas, reducing effort while maximizing risk coverage.
**Explanation:** Risk‑based testing focuses resources where defects would be most costly.

11️⃣ *What does QA stand for in software development?*
   - A) Quality Assurance
   - B) Quick Access
   - C) Quality Analysis
   - D) Query Algorithm
**Answer:** Quality Assurance
**Explanation:** QA refers to the systematic process of ensuring software quality through planned activities.

12️⃣ *Which ISO standard is commonly associated with software quality?*
   - A) ISO 9001
   - B) ISO 14001
   - C) ISO 27001
   - D) ISO 50001
**Answer:** ISO 9001
**Explanation:** ISO 9001 sets criteria for a quality management system applicable to software organizations.

13️⃣ *Which methodology emphasizes iterative development and customer collaboration?*
   - A) Waterfall
   - B) Agile
   - C) Spiral
   - D) V-Model
**Answer:** Agile
**Explanation:** Agile emphasizes iterative cycles, customer feedback, and adaptability over rigid planning.

14️⃣ *Which of the following is NOT a characteristic of manual testing?*
   - A) Exploratory
   - B) Requires no code
   - C) Fast execution
   - D) Human judgment
**Answer:** Fast execution
**Explanation:** Manual testing relies on human judgment and is typically slower than automated tests.

15️⃣ *What is the main advantage of manual testing?*
   - A) Speed
   - B) Automation
   - C) Human intuition
   - D) Low cost
**Answer:** Human intuition
**Explanation:** Manual testing allows exploratory testing and nuanced human judgment.

16️⃣ *Which technique measures the percentage of code exercised by tests?*
   - A) Code coverage
   - B) Cyclomatic complexity
   - C) Static analysis
   - D) Performance profiling
**Answer:** Code coverage
**Explanation:** Code coverage quantifies which lines, branches, or paths are executed during testing.

17️⃣ *Which tool is commonly used for UI automation in web applications?*
   - A) Selenium
   - B) Docker
   - C) Git
   - D) Jenkins
**Answer:** Selenium
**Explanation:** Selenium drives browsers to automate UI interactions.

18️⃣ *Define regression testing.*
**Answer:** Running previously passed tests to ensure new changes haven't broken existing functionality.
**Explanation:** Regression testing verifies that recent code changes do not adversely affect existing features.

19️⃣ *Manual testing can be fully automated with scripts.*
**Answer:** False
**Explanation:** Certain exploratory and usability aspects require human testers.

20️⃣ *Which process model combines iterative development with risk analysis?*
   - A) Waterfall
   - B) Agile
   - C) Spiral
   - D) V-Model
**Answer:** Spiral
**Explanation:** The Spiral model combines prototyping and risk analysis in iterative cycles.
